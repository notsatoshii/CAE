    **A) `components/memory/why-drawer.tsx`** (client):

    Props:
    ```typescript
    interface WhyDrawerProps {
      open: boolean;
      onClose: () => void;
      taskId: string | null;              // null while no task is selected
      filesModified?: string[];           // optional; if provided, powers heuristic fallback
    }
    ```

    Render logic:
    - Closed (`!open` or `!taskId`) → return `null`.
    - Open:
      - Panel slides from the right (Tailwind transition — matches NodeDrawer from plan 08-05 visually).
      - Header: "Why? · task {taskId}" + close button (ESC + click-outside both wired).
      - Explain tooltip: `memoryExplainWhy` copy.
      - Body: 3 mutually-exclusive render paths based on fetched state.

    Fetch on `open + taskId` change:
    1. `GET /api/memory/consult/${encodeURIComponent(taskId)}`
    2. On 200: `setResult({ result: MemoryConsultResult, error: null })`.
    3. On non-200: `setResult({ result: null, error: "..." })` → render `memoryLoadFailed` copy.

    Render paths:

    **Path A — Live trace (real):**
    When `result?.found === true`:
    - Pill with `memoryWhyLiveTracePill` (cyan, small).
    - List of entries sorted by `ts` ascending; each row: file basename (click → emits `onSelectFile(absPath)` — optional prop, Wave 5 wires to BrowseTab selection), full abs path (mono, muted), timestamp (human-formatted via existing `formatRelativeTime` if present; otherwise `new Date(ts).toLocaleString()`).

    **Path B — Heuristic fallback:**
    When `result?.found === false` AND `filesModified && filesModified.length > 0`:
    - Compute `heuristic = getHeuristicWhyTrace(filesModified)`.
    - If `heuristic.length > 0`:
      - Pill with `memoryWhyHeuristicPill` (amber/warning styling, small).
      - List: same rendering as Path A, but rows show no `ts` (instead display a dev-mode-only note "basis: files_modified_intersect").
      - Sub-note below list in dev-mode only: "Install the PostToolUse hook to capture real traces (docs/memory-hook.md)".

    **Path C — Empty:**
    When `result?.found === false` AND (no `filesModified` OR heuristic is empty):
    - Center-aligned muted message: `memoryWhyEmpty` copy.

    Error path: below all three, if `error`, show error banner.

    **B) `components/memory/why-drawer.test.tsx` (Vitest + RTL, 5 cases):**

    1. Path A — fetch returns `{ found: true, entries: [ {source_path: "/x/AGENTS.md", ts: "2026-04-22T00:00:00Z"} ] }` → renders "Live trace" pill + entry row with "/x/AGENTS.md".
    2. Path B — fetch returns `{ found: false, entries: [] }` + prop `filesModified: ["/x/AGENTS.md", "/x/foo.txt"]` → renders "Heuristic" pill + "/x/AGENTS.md" only (foo.txt filtered out).
    3. Path C — fetch returns `{ found: false, entries: [] }`, no filesModified → renders empty-state copy.
    4. Fetch 500 → error banner shown.
    5. ESC keypress → calls `onClose`.

    Stub `fetch` per test; use `vi.mock("@/lib/cae-memory-whytrace", ...)` to control heuristic returns.

    **C) Constraints:**
    - `"use client"`.
    - No imports from `components/memory/browse/*` or `graph/*` — drawer is shared.
    - Uses `useEffect` for fetch + cleanup on close.
    - A11y: role="dialog" + aria-labelledby on header + focus-trap while open.
    - No USD copy (follow memory-hygiene even though it's tokens-unrelated).
