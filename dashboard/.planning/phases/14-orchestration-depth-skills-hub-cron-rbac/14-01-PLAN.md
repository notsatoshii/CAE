<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dev machine → npm registry | Installing third-party packages; tamper risk via malicious dep |
| dev machine → gitleaks binary host | Downloading gitleaks binary from GitHub release |
| shell env → audit-hook.sh | PostToolUse hook receives CAE_TASK_ID and CLAUDE_TOOL_NAME from hook caller; no user input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01-01 | Tampering | install-gitleaks.sh binary download | mitigate | Pin version 8.18.4 explicitly; prefer Homebrew when available (signed); fall back to HTTPS GitHub release. TODO v2: verify checksum |
| T-14-01-02 | Injection | audit-hook.sh jq args | mitigate | Use `jq -nc --arg` with typed args (never shell string interpolation); printf fallback uses `%s` with shell-quoted vars — cwd escape risk remains low because PWD is always a filesystem path |
| T-14-01-03 | Tampering | scheduled_tasks.json at repo root | accept | File is .gitignored; real protection is in Plan 14-03 where we add file permissions 0600 + Zod schema validation before watcher consumption |
| T-14-01-04 | Info Disclosure | gitleaks-report.json fixture containing AKIA... | accept | Secret value is literal "REDACTED"; fixture is for parser tests only, never a real key |
</threat_model>
