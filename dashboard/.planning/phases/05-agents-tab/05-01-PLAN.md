    **Implementation:**

    1. **Data-collection pass** — single async helper `collectAllEvents()`:
       - Call `listProjects()` from `@/lib/cae-state`
       - For each project, `tailJsonl(join(project.path, ".cae", "metrics", "circuit-breakers.jsonl"), 5000)`
       - Return `Array<{project: string, projectPath: string, event: Record<string, unknown>}>` (flat list, all events across projects)
       - Wrap individual project reads in try/catch; skip failing projects (log to console.error)

    2. **Per-agent bucket pass** — for each of the 9 agents in `AGENT_META`:
       - Filter events where `(entry.event.agent ?? "forge").toLowerCase() === agentName` (default "forge" per Phase 4 convention)
       - Partition by recency: 7d window, 30d window, 24h window, 30s concurrent window
       - **Bucketing for sparklines (10 buckets over 7d):**
         - `bucket_ms = 7 * 86400000 / 10 = 60480000`
         - `now = Date.now()`; bucket index = `Math.floor((now - eventTs) / bucket_ms)`; keep 0..9
         - For tokens_per_hour: `sum((inputTokens||0) + (outputTokens||0)) / (bucket_ms / 3600000)` per bucket
         - For success_history: `(count of forge_done in bucket) / max(1, count of forge_done + forge_fail in bucket)` — produces 0..1 per bucket
         - For wall_history: mean wallMs of completed events per bucket; 0 if empty
       - Totals (tokens_total / success_rate / avg_wall_ms) = flat aggregates over 7d window using only events with defined tokens/wallMs
       - 30d baseline: success_rate_30d = completed_success_30d / max(1, completed_total_30d)

    3. **Drift detection** — `const THRESHOLD = 0.85;`
       - `drift_warning = (samples_7d >= 5) && (success_rate_30d > 0) && (success_rate_7d < success_rate_30d * THRESHOLD)`
       - Never flag dormant-waking agents (the `samples_7d >= 5` gate handles this).

    4. **Concurrency + group classification:**
       - `concurrent` = 30s sliding window: count of `forge_start` events for this agent in last 30000ms with no matching `forge_done`/`forge_fail` within the same window (match by taskId; fall back to order-based pairing if taskId missing — mirror Phase 4 04-01 approach in `cae-home-state.ts:buildGlobalActiveAgents`).
       - `queued` = number of outbox entries assigned to this agent (if any — fall back to 0; this is UI-polish detail, not blocking).
       - `last_24h_count` = count of all events for this agent with ts within 24h.
       - `last_run_days_ago` = `Math.floor((now - lastEventTs) / 86400000)`; `null` if zero events ever.
       - `group`:
         - `"active"` if `concurrent > 0`
         - else `"recently_used"` if `last_run_days_ago !== null && last_run_days_ago <= 7`
         - else `"dormant"`

    5. **Model resolution** — read `last known model` per agent:
       - Walk events newest-to-oldest, return first `entry.event.model` encountered; fallback `"claude-sonnet-4-6"`.

    6. **Process-level cache** — mirror `cae-home-state.ts` pattern but at 30s TTL (per CONTEXT §Metrics aggregation):
       ```typescript
       const CACHE_TTL_MS = 30_000;
       let cached: { at: number; value: Awaited<ReturnType<typeof buildRoster>> } | null = null;
       ```
       Both `getAgentsRoster()` and `getAgentDetail(name)` share the same underlying roster build; detail layers persona_md + lifetime + invocations on top.

         - Use `CAE_ROOT` from `@/lib/cae-config`.
       - Lifetime stats: iterate full event set (not just 7d) for this agent.
         - `top_expensive` = top 5 events by `inputTokens + outputTokens`, map to `{project, phase, plan, task, tokens, timestamp}`. `phase` = `event.phaseId ?? ""`; `plan` = derive from phaseId if present (e.g. split on `-pl`), else `""`; `task` = `event.taskId ?? ""`.
       - `recent_invocations` = last 50 events (sorted DESC by ts) mapped to AgentInvocation shape.
         - `status: "ok"` for `forge_done`, `"fail"` for `forge_fail`; skip `forge_start`/`retry`/`phantom_escalation` from this list.

    **A) `dashboard/components/ui/sparkline.tsx`** — hand-rolled SVG primitive (NO chart library).

    ```tsx
    import * as React from "react"
    import { cn } from "@/lib/utils"

    interface SparklineProps {
      values: number[]
      width?: number
      height?: number
      color?: string
      className?: string
      ariaLabel?: string
    }

    export function Sparkline({
      values,
      width = 120,
      height = 24,
      color = "var(--accent, #00d4ff)",
      className,
      ariaLabel,
    }: SparklineProps) {
      if (!values || values.length === 0) {
        return (
          <svg
            width={width}
            height={height}
            className={cn("block", className)}
            aria-hidden={ariaLabel ? undefined : true}
            aria-label={ariaLabel}
            role={ariaLabel ? "img" : undefined}
            data-testid="sparkline-empty"
          />
        )
      }
      const min = Math.min(...values)
      const max = Math.max(...values)
      const range = max - min || 1
      const step = values.length > 1 ? width / (values.length - 1) : 0
      const points = values
        .map((v, i) => {
          const x = i * step
          const y = height - ((v - min) / range) * height
          return x.toFixed(2) + "," + y.toFixed(2)
        })
        .join(" ")
      return (
        <svg
          width={width}
          height={height}
          viewBox={"0 0 " + width + " " + height}
          className={cn("block", className)}
          aria-hidden={ariaLabel ? undefined : true}
          aria-label={ariaLabel}
          role={ariaLabel ? "img" : undefined}
          data-testid="sparkline"
        >
          <polyline
            points={points}
            fill="none"
            stroke={color}
            strokeWidth={1.5}
            strokeLinecap="round"
            strokeLinejoin="round"
          />
        </svg>
      )
    }
    ```

    Requirements:
    - Pure component, no hooks, no client directive needed (renders identically on server and client; may be imported by client components without "use client" forcing).
    - Default width 120px, height 24px, 1.5px stroke, cyan accent.
    - Graceful empty: returns an empty `<svg>` of the requested size when `values` is empty (prevents layout shift).
    - Accessibility: `aria-hidden` by default (sparklines convey a summary also shown as text); allow `ariaLabel` opt-in for non-redundant cases.
