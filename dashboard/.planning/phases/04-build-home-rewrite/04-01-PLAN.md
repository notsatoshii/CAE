The extended GET /api/state response shape (server returns; client consumes via useStatePoll()):

```typescript
// lib/cae-home-state.ts exports
export interface Rollup {
  shipped_today: number;      // count of outbox DONE.md entries (status=success) OR merged commits today
  tokens_today: number;       // sum of inputTokens + outputTokens across all projects' circuit-breakers.jsonl entries today
  in_flight: number;          // count of phases with at least one active (running) task across all projects
  blocked: number;            // count of tasks with sentinel_reject >=3 OR status=failed in circuit-breakers.jsonl within last 24h
  warnings: number;           // count of phantom_escalation events within last 24h
}

export interface AgentActive {
  name: string;               // "forge" | "sentinel" | "scout" | "scribe" | "phantom" | "aegis" | "arch" | "herald" | "nexus"
  concurrent: number;         // number of tasks this agent is currently executing (for avatar dots)
}

export interface PhaseSummary {
  project: string;            // project path (e.g. /home/cae/ctrl-alt-elite/dashboard)
  projectName: string;        // derived short name (e.g. "cae-dashboard")
  phase: string;              // phase dir name (e.g. "04-build-home-rewrite")
  phaseNumber: number;        // integer parsed from prefix
  wave_current: number;       // highest wave with at least one running task (or last completed + 1)
  wave_total: number;         // max wave number across plan files
  progress_pct: number;       // rounded 0..100 — (merged_tasks / total_tasks) * 100
  eta_min: number | null;     // heuristic: (avg_wall_time_prev_phases * remaining_waves); null if no prior history
  tokens_phase: number;       // sum of tokens in circuit-breakers.jsonl entries whose phaseId matches this phase dir
  agents_active: AgentActive[]; // agents with at least one running task in this phase
}

export interface RecentEvent {
  ts: string;                 // ISO 8601 timestamp
  project: string;            // project path
  projectName: string;        // short name
  phase: string;              // phase dir name
  plan: string;               // plan id (e.g. "p3-pl03" or "04-01")
  status: "shipped" | "aborted";
  commits: number;            // count of new commits landed for this task
  agent: string;              // lowercase agent name
  model: string;              // e.g. "sonnet" | "opus" | "haiku"
  tokens: number;             // total tokens this event consumed
}

export interface NeedsYouItem {
  type: "blocked" | "dangerous" | "plan_review";
  project: string;            // project path
  projectName: string;        // short name
  phase?: string;             // phase dir name (absent for plan_review)
  task?: string;              // task id (e.g. "p4-pl03-t1")
  summary: string;            // one-line human description (e.g. "Sentinel rejected 3×")
  actions: Array<{ label: string; href: string }>; // e.g. [{ label: "Review", href: "/build/phase/4?sheet=open&plan=p4-pl03&task=t1" }]
}

export interface HomeState {
  rollup: Rollup;
  phases: PhaseSummary[];
  events_recent: RecentEvent[]; // last 20 across all projects, newest first
  needs_you: NeedsYouItem[];
  live_ops_line: string;       // pre-formatted per CONTEXT §Live Ops line composition
}

export function getHomeState(): Promise<HomeState>;
```

<live_ops_composition>
<!-- LOCKED by CONTEXT §Live Ops line composition. Server composes this string; UI renders verbatim. -->

Given the list of `AgentActive` records with running tasks:
- 0 active agents: `"Idle right now."`
- 1 active agent: `"Right now: {founder_label} is on {task}."` (e.g., `"Right now: the builder is on p4-pl03-t1."`)
- 2 active agents: `"Right now: {a_founder_label} is on {task_a}. {b_founder_label} is on {task_b}."`
- 3+ active agents: show first 3 active agents and append `" +{N} more idle"` if more

Note: founder_label comes from the agent-meta lookup in Plan 04-02 (`the builder`, `the checker`, etc.). For this plan, the server uses a minimal inline map (`forge→the builder`, `sentinel→the checker`, `scout→the researcher`, `scribe→the memory-keeper`, others→agent name) — Plan 04-02 centralizes this. DO NOT DEPEND on agent-meta here (Plan 04-02 runs in parallel); use the inline map in this plan and Plan 04-02 refactors server usage if needed.

Task identifier format: `p{phaseNumber}-pl{plan_id}-t{task_id}` (e.g. `p4-pl03-t1`). For agents with no identifiable task, use `"idle"`.
</live_ops_composition>
