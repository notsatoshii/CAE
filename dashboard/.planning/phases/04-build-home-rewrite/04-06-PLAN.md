    ```bash
    # Crude but effective check: for every file in build-home/, for every <button open-tag, ensure
    # either (a) the tag has aria-label= OR (b) the button's inner content includes non-whitespace,
    # non-tag text.
    echo ""
    echo "=== Button accessible-name audit ==="
    for f in dashboard/components/build-home/*.tsx; do
      # Find all <button ...> open tags (could span lines — approximate with grep)
      BUTTON_COUNT=$(grep -cE '<button\b' "$f")
      ARIA_LABEL_COUNT=$(grep -cE '<button\b[^>]*aria-label=' "$f")
      # Text-bearing buttons: <button ...>Some text</button> pattern OR
      # <button ...>{t.someLabel}</button> pattern
      # grep for button tag followed (on same or next few lines) by >text< pattern
      # Simpler heuristic: count button tags that contain "aria-label" OR are followed by
      # a JSX text/expression node within the next 5 lines
      TEXT_BUTTONS=$(awk '/<button\b/{in_btn=1; buf=""} in_btn{buf=buf"\n"$0; if($0 ~ /<\/button>/){in_btn=0; if(buf ~ /aria-label=/ || buf ~ />[^<]*[[:alnum:]][^<]*</  || buf ~ />\{[^}]*\}</){print "ok"}}}' "$f" | wc -l)
      if [ "$BUTTON_COUNT" -gt 0 ] && [ "$TEXT_BUTTONS" -eq 0 ]; then
        echo "WARN: $f has $BUTTON_COUNT <button> tags but 0 with text/aria-label — manual review"
      else
        echo "OK: $f buttons=$BUTTON_COUNT text_or_aria=$TEXT_BUTTONS"
      fi
    done
    ```

    The awk pattern matches a `<button>` open tag, buffers lines until `</button>`, then checks if the buffered block contains `aria-label=` OR a JSX text node (`>SOMETEXT<`) OR a JSX expression (`>{...}<`). Any button failing all three is flagged for manual review. If any WARN line appears, address the specific button (add aria-label or visible text) before proceeding.
