<open_questions>
**Q1 — Should the legend render INSIDE the canvas (drawn by renderer) or as a floating `<aside>` beside it?**
Decision: **floating `<aside>`** (the `FloorLegend` component from Plan 02). The renderer already draws per-station text labels; a separate DOM-based legend is better for screen-readers (Explain-mode fallback). Position: bottom-right, 240px wide, translucent surface bg.

**Q2 — Project resolution when no `?project=` search param is provided: use the most recent Shift project, or `process.cwd()`?**
Decision: **most recent Shift project** (from `listProjects()` sorted by `shiftUpdated` desc — Plan 10 already delivers this). Falls back to `process.cwd()` if no Shift projects exist. Consistent with Plan-home behavior.

**Q3 — Toolbar position: top-right overlay on the canvas, or fixed bottom bar?**
Decision: **top-right overlay** inside the page, absolute-positioned over the canvas. Matches UI-SPEC §11 "Controls: top-right pop-out button, minimize, pause animation."

**Q4 — Does `popout=true` hide the top-nav chrome entirely?**
Decision: for this plan's scope, `popout=true` hides ONLY the FloorToolbar's "Pop-out" button and SHOWS the "Minimize" button. The actual `popout=1` full-chrome-hide (removing TopNav, hiding CostTicker, etc.) is **Plan 05**'s responsibility. This plan gets popout-aware toolbar behavior; Plan 05 layers on the route-level chrome suppression.
</open_questions>
