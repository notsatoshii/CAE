<open_questions>
**Q1 — Chrome suppression: route group, layout swap, or CSS hide?**

Options considered:
- **Option A: Route group `(popout)`** — Next.js route groups can have their own layout. Would require restructuring `app/floor/` into `app/(popout-group)/floor/popout/` with its own `layout.tsx` NOT wrapping TopNav. Big refactor.
- **Option B: Layout-level conditional** — `app/floor/layout.tsx` checks pathname via `headers()` or a client-side detector; renders without TopNav when path ends in `/popout`. Requires server-side pathname inspection.
- **Option C (chosen): Route-scoped CSS** — `app/floor/popout/page.tsx` emits a `<style>` tag hiding `header[data-testid="top-nav"]`. Simple, reversible, tested via querySelector absence.

**Decision: Option C.** Pragmatic. Zero refactor. The chrome isn't "gone" — it's `display: none` within the popout route only (CSS scope: no global side-effect because the `<style>` is inside the popout page's JSX subtree — React mounts/unmounts it with the route).

Tradeoff: screen readers still see the hidden nav in the accessibility tree unless we add `aria-hidden`. Address by setting `aria-hidden="true"` AND `display: none` together — covered in Task 1 implementation.

**Q2 — Pop-out URL: `/floor?popout=1` (Plan 04) vs `/floor/popout` (this plan)?**

Plan 04 used `/floor?popout=1` because the dedicated route didn't exist yet. **This plan switches to `/floor/popout`** — cleaner routing, easier chrome scope, easier bookmarking. Plan 04's toolbar test gets updated (Task 1 action).

**Q3 — Should the popout be allowed at all when user is not authenticated?**
Yes, guarded identically to /floor — middleware + page-level auth() + redirect. Window.open on unauthenticated session → server responds with redirect → child window sees the signin page. User signs in there → subsequent requests authed.

**Q4 — What if the user bookmarks `/floor/popout?project=/some/path` and navigates there directly (not via pop-out button)?**
Works fine: `window.opener === null`, so the return-to-main-window link is hidden (per spec), Escape-to-close is disabled (window is user-owned, not opener-owned), rest of the page functions normally. Same URL is usable as a bookmarked dedicated dashboard tab.
</open_questions>

       "use client";
       import { useEffect } from "react";
       import FloorClient from "./floor-client";
       import { useDevMode } from "@/lib/providers/dev-mode";
       import { labelFor } from "@/lib/copy/labels";

       export interface FloorPopoutHostProps {
         cbPath: string | null;
         projectPath: string | null;
       }

       export function FloorPopoutHost({ cbPath, projectPath }: FloorPopoutHostProps) {
         const { dev } = useDevMode();
         const L = labelFor(dev);

         useEffect(() => {
           const prevTitle = document.title;
           document.title = L.floorPageTitle + " — pop out";
           // Best-effort size nudge — browsers may ignore.
           if (typeof window !== "undefined" && window.opener) {
             try { window.resizeTo(960, 720); } catch { /* ignore */ }

           return () => { document.title = prevTitle; };

         useEffect(() => {

           if (window.opener == null) return; // only bind when we're actually a popout
           const onKey = (e: KeyboardEvent) => {
             if (e.key === "Escape") {

           document.addEventListener("keydown", onKey);
           return () => document.removeEventListener("keydown", onKey);
         }, []);

         return <FloorClient cbPath={cbPath} projectPath={projectPath} popout={true} />;
       }

       export default FloorPopoutHost;
